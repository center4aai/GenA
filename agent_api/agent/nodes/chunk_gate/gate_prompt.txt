### ЖЁСТКИЕ ПРАВИЛА ВЫВОДА

- ВЫВОД ДОЛЖЕН БЫТЬ ВАЛИДНЫМ JSON.
- НЕЛЬЗЯ ИСПОЛЬЗОВАТЬ MARKDOWN (НИКАКИХ ```).
- НИКАКИХ КОММЕНТАРИЕВ ВНЕ JSON.
- НИКАКОГО ДОПОЛНИТЕЛЬНОГО ТЕКСТА ДО ИЛИ ПОСЛЕ JSON.
- НИКАКИХ ПЕРЕНОСОВ СТРОК ВНЕ САМОЙ JSON-СТРУКТУРЫ.
- ИМЕНА КЛЮЧЕЙ ДОЛЖНЫ СОВПАДАТЬ ТОЧНО (ВКЛЮЧАЯ ПРОБЕЛЫ И ПОДЧЁРКИВАНИЯ).
- ЗНАЧЕНИЯ БИНАРНЫХ ПОЛЕЙ ДОЛЖНЫ БЫТЬ В ВИДЕ СПИСКА: [0] ИЛИ [1].
- `confidence` ДОЛЖЕН БЫТЬ ВЕЩЕСТВЕННЫМ ЧИСЛОМ В ДИАПАЗОНЕ 0.0–1.0.
- ИСПОЛЬЗУЙ ДРОБНЫЕ ЗНАЧЕНИЯ (например, 0.83, 0.91), А НЕ ТОЛЬКО 0.5 ИЛИ 1.0.
- НЕ СТАВЬ ВЫСОКУЮ УВЕРЕННОСТЬ ПО УМОЛЧАНИЮ.
- ДЛЯ НЕОДНОЗНАЧНЫХ СЛУЧАЕВ СНИЖАЙ `confidence`.
- `reasoning` ДОЛЖЕН БЫТЬ РОВНО ОДНИМ КОРОТКИМ ПРЕДЛОЖЕНИЕМ ДЛЯ КАЖДОГО КРИТЕРИЯ.

НЕЛЬЗЯ:

- ВЫВОДИТЬ ТЕКСТ ВРОДЕ:
"Here is the result:"
"Explanation:"
- МЕНЯТЬ ИМЕНА КЛЮЧЕЙ.
- ВЫВОДИТЬ:
0, 1 (БЕЗ СПИСКА),
"1" (СТРОКА).
- ДОБАВЛЯТЬ ЛИШНИЕ ПОЛЯ.
- ПРОПУСКАТЬ КАКОЕ-ЛИБО ПОЛЕ.
- ФОРМАТИРОВАТЬ ОТВЕТ КАК MARKDOWN.
- СТАВИТЬ ВЫСОКУЮ УВЕРЕННОСТЬ (>= 0.9) ДЛЯ НЕОДНОЗНАЧНЫХ СЛУЧАЕВ.
- ВЫХОДИТЬ ЗА ДИАПАЗОН 0.0–1.0 ДЛЯ `confidence`.
- ИСПОЛЬЗОВАТЬ ТОЛЬКО ДИСКРЕТНЫЕ ЗНАЧЕНИЯ (0.5 И 1.0) ДЛЯ ВСЕХ СЛУЧАЕВ.
